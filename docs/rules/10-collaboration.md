# Collaboration Rules

## Working Mode

Use vibe coding, but keep it disciplined:

- Clarify intent first.
- Write down rules before major implementation.
- Build in small increments.
- Verify after each meaningful change.
- Keep customer-facing quality above technical showmanship.

## Before Coding

Check:

1. Product decision impact
2. Mobile impact
3. Visual tone impact
4. Data model impact
5. Deployment impact
6. Acceptance criteria

## During Coding

- Prefer simple, maintainable implementation.
- Keep real project needs above tutorial-style patterns.
- Use stable file structure.
- Avoid introducing backend complexity into MVP unless requested.
- Keep user-provided assets organized.
- Do not remove user files without explicit need.

## Communication

When a choice affects product direction, ask or state a sensible default.

When a choice is technical and low risk, choose conservatively and document it.

## Verification

At minimum, each phase should verify:

- Build or syntax checks
- Mobile layout
- Main user flow
- No customer-facing development language
- Git status summary
