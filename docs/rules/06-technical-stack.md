# Technical Stack

## Stage A: Frontend MVP

Recommended:

```text
React
TypeScript
Vite
Lucide React icons
Tailwind CSS or CSS Modules
GitHub Pages
```

Why:

- Fast to build and share.
- Helpful for frontend and full-stack job search.
- Keeps MVP simple.
- Can later migrate to Next.js or connect to a backend.

## Stage B: Formal Full-Stack Version

Recommended:

```text
Next.js
React
TypeScript
Java
Spring Boot
Spring Data JPA
PostgreSQL
DigitalOcean Spaces
Nginx
Docker Compose
```

Why:

- React / TypeScript supports frontend and full-stack roles.
- Spring Boot / PostgreSQL helps Java backend job readiness.
- Docker / Nginx / DigitalOcean helps real deployment experience.

## Stage C: Job-Value Enhancements

Add after the core product is useful:

```text
Redis
RabbitMQ
Kafka
JUnit
Testcontainers
Actuator
GitHub Actions
```

These should not block MVP.

## Engineering Principle

Use the simplest stack that can deliver the current phase, but keep the architecture compatible with the formal Java backend phase.
